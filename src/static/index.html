Authentication Methods

HTTP Basic Authentication

Sends credentials base64-encoded in the Authorization: Basic <blob> header.

How it works:
  • Client hashes: b64encode(username + ":" + password)
  • Sends header: Authorization: Basic YWxpY2U6cGFzc3dvcmQ=
  • Stateless: Every single request must include this header.

HTTP Digest Authentication (RFC 2617)

Challenge-response scheme. Server challenges with a nonce. Client responds with MD5 hashes, protecting the password from wire-sniffing.

Double Request Handshake:
  1. Client requests /auth/digest/protected without auth.
  2. Server replies with 401 Unauthorized & WWW-Authenticate: Digest realm="...", nonce="..."
  3. Client calculates MD5 hashes using the nonce, realm, and password.
  4. Client sends secondary request with Authorization: Digest username="...", response="..."

API Key Authentication

Header-based token auth. Use Basic credentials to generate an API key, then use the key to authenticate.

Your New API Key (Save it!): ap_...
How it works:
  • Client sends key in custom header: X-API-Key: ap_xyz...
  • Server hashes key via SHA-256 and checks database.
  • Commonly used for Machine-to-Machine (M2M) communication.

Cookie-Based Session Authentication

Stateful authentication. Server validates credentials once, saves session state, and returns a session cookie. Browser automatically sends cookies on subsequent requests.

Stateful session lifecycle:
  • Login: Server validates credentials, generates random session ID, records it in the store, and returns Set-Cookie: auth2prod_session_id=...
  • Requests: Browser automatically attaches Cookie: auth2prod_session_id=...
  • Scale problem: In-Memory stores fail when requests go to different server instances. Database/Redis stores solve this by centralizing state.
